NewModelsPage

NewModelsPage class

A Flutter screen that displays a list of available (dummy) models fetched from a backend server, allowing users to activate/select a model.

Purpose:

  • NewModelsPage is intended to let users view pre-defined models from the backend, typically for selection or testing purposes.
  • Each model can be in an Active or Inactive state, and users can trigger activation.

API Integration:

GET Available Models

  • Endpoint:
    GET https://services.iittp.ac.in/var/homes/<homeId>/api/dummy_models/
  • Headers:
    { "Authorization": "Api-Key <apiKey>" }
  • Response Format: 
    {
  "models": [
    {
      "model_id": "uuid",
      "model_name": "IntruderAlertModel",
      "status": true
    },
    ...
  ]
}

POST Select/Activate Model

  • Endpoint:
    POST https://services.iittp.ac.in/var/homes/<homeId>/api/dummy_model_activate/

  • Body:

    { "model_id": "<uuid>" }

  • Headers:

    • Authorization: Api-Key
    • Content-Type: application/json

  • Response Example:

    { "message": "Model activated successfully" }

State Variables:

  • models: List of models fetched from the server.
  • isLoading: Boolean to show a loading spinner while data is fetched.

Methods:

  • fetchNewModels()

    • Fetches dummy models via GET request using homeId and apiKey from DatabaseHelper.
    • Sets the models list and handles loading state.

  • selectModel(modelId)

    • Sends POST request to activate a selected model by ID.
    • Shows success/error via SnackBar.

  • showSnack(msg, {isError})

    • Helper method to show feedback to the user.

UI Structure:

  • AppBar titled "Available Models"
  • Body:
    • While loading → CircularProgressIndicator
    • If no models → "No models found"
    • Else:
      • A scrollable ListView of model cards.
      • Each card includes:
        • Model name
        • Status (Active/Inactive) with color
        • Select button to activate model

Visuals:

  • Cards have rounded corners and elevation.
  • Active status is colored green; inactive is red.
  • Button uses theme's primary color.

Used In:

  • Typically accessible via a navigation option like “Explore Models” or “New Models”.

Future Suggestions:

  • Add search/filter capabilities by model name or tag.
  • Show model descriptions and icons.
  • Support pagination if model list grows large.
  • Add model preview or testing feature.
