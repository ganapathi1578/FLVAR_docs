ModelDetailPage

ModelDetailPage class

A detailed Flutter page for viewing AI/ML model metadata and downloading model versions.

API Service:

  1. Fetch Model Details

    • Endpoint: 
      POST https://services.iittp.ac.in/var/homes/<homeId>/trainingspace/model_details/
    • Headers:
      • Authorization: Api-Key <apiKey>
      • Content-Type: application/json
    • Body: 
      { "model_id": "<modelId>" }
    • Response: 
      {
  "model_id": "abc123",
  "name": "ActionNet v2",
  "upload_date": "2024-06-01",
  "tag": "Surveillance",
  "visibility": "public",
  "vendor_userid": "john_doe",
  "description": "A deep model for action detection...",
  "versions": [
    {
      "version_id": "v1.0.0",
      "version_name": "Initial Release",
      "description": "Original version.",
      "upload_date": "2024-06-01",
      "size": 14956000,
      "compatibility": "Flutter v3.22"
    },
    ...
  ]
}

  2. Download / Switch Model Version

    • Endpoint: 
      POST https://services.iittp.ac.in/var/homes/<homeId>/containers/switch/
    • Body: 
      {
  "model_id": "<modelId>",
  "version_id": "<versionId>"
}
    • Success Response: 200 OK
      • Triggers success snack: "Model version switched successfully!"
    • Failure:
      • Parses body['error'] if provided.

Fields:

  • modelId: Required to fetch and switch versions for a specific model.
  • modelData: JSON of model metadata (e.g., name, vendor, description).
  • versions: List of version maps, each containing size, upload date, compatibility, etc.
  • isLoading: Boolean flag to show loading spinner.
  • _showFullDescription: Toggles truncation of long descriptions.

Methods:

  • fetchModelDetails()
    • Fetches model metadata and available versions from the API.
  • downloadVersion(modelId, versionId)
    • Sends version switch request.
  • formatDate()
    • Converts ISO date strings to readable format like "July 8, 2025".
  • formatBytes()
    • Converts byte size to KB, MB, or GB.
  • showSnack()
    • Displays success/error message via SnackBar.
  • detailRow(label, value)
    • UI helper to display key-value text rows.

UI Behavior:

  • Displays model name, metadata (upload date, tag, visibility).
  • Description section is truncatable with “Show more / Show less”.
  • Below the description, a scrollable list of expandable cards for each version:
    • ExpansionTile contains:
      • Version name, size, and upload date in the header.
      • Description, compatibility, download button in expanded section.

User Action:

  • Tap "Download" on a version to trigger version switch via API.

Note:

  • Version switching assumes the model is containerized (e.g., for deployment).
  • Robust error handling ensures that network/API errors show a SnackBar alert.
  • Uses DatabaseHelper to securely retrieve saved homeId and apiKey from SQLite.

Used in:

  • Navigated to from MarketplacePage on model card tap.
  • Can be extended to support live preview, logs, or permissions control.
